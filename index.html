3. El Game Loop (Bucle del Juego)

¬øQu√© es el Game Loop?

El Game Loop es el coraz√≥n de cualquier videojuego. Es un bucle que se ejecuta constantemente (idealmente 60 veces por segundo) y realiza tres tareas principales:

üîÑ Las 3 Fases del Game Loop:
  1. Input (Entrada): Capturar eventos del usuario (teclado, mouse)
  2. Update (Actualizar): Actualizar la l√≥gica del juego (mover objetos, calcular colisiones)
  3. Render (Dibujar): Dibujar todo en la pantalla

Estructura del Game Loop

import pygame
import sys

pygame.init()
pantalla = pygame.display.set_mode((800, 600))
pygame.display.set_caption("Game Loop B√°sico")

# Clock para controlar FPS
reloj = pygame.time.Clock()
FPS = 60  # Frames por segundo

# Colores
NEGRO = (0, 0, 0)
BLANCO = (255, 255, 255)

corriendo = True
while corriendo:
    # ===== FASE 1: INPUT (Eventos) =====
    for evento in pygame.event.get():
        if evento.type == pygame.QUIT:
            corriendo = False
        if evento.type == pygame.KEYDOWN:
            if evento.key == pygame.K_ESCAPE:
                corriendo = False
    
    # ===== FASE 2: UPDATE (L√≥gica del juego) =====
    # Aqu√≠ actualizamos posiciones, calculamos colisiones, etc.
    
    # ===== FASE 3: RENDER (Dibujar) =====
    pantalla.fill(NEGRO)  # Limpiar pantalla
    
    # Aqu√≠ dibujamos todo
    
    pygame.display.flip()  # Actualizar pantalla
    
    # ===== CONTROLAR FPS =====
    reloj.tick(FPS)  # Esperar para mantener 60 FPS

pygame.quit()
sys.exit()

¬øQu√© es FPS?

FPS (Frames Per Second) indica cu√°ntas veces por segundo se actualiza la pantalla. 60 FPS significa que el game loop se ejecuta 60 veces por segundo.

FPS Descripci√≥n Uso
30 FPS Fluido para juegos casuales Juegos de estrategia, puzzles
60 FPS Est√°ndar para juegos modernos La mayor√≠a de juegos 2D
120+ FPS Ultra fluido Juegos competitivos
üí° El m√©todo tick():

reloj.tick(60) hace que el programa espere el tiempo necesario para mantener 60 FPS. Si el c√≥digo se ejecuta muy r√°pido, a√±ade una pausa. Si es demasiado lento, permite que el juego contin√∫e.

6. Manejo de Eventos

¬øQu√© son los Eventos?

Los eventos son acciones que ocurren durante el juego: presionar una tecla, mover el mouse, hacer clic, etc. Pygame mantiene una cola de eventos que debemos revisar en cada frame.

Tipos de Eventos Principales

Tipo de Evento Cu√°ndo Ocurre Atributos √ötiles
QUIT Usuario cierra la ventana -
KEYDOWN Se presiona una tecla evento.key
KEYUP Se suelta una tecla evento.key
MOUSEBUTTONDOWN Se presiona bot√≥n del mouse evento.pos, evento.button
MOUSEBUTTONUP Se suelta bot√≥n del mouse evento.pos, evento.button
MOUSEMOTION Se mueve el mouse evento.pos, evento.rel

Eventos del Teclado

import pygame
import sys

pygame.init()
pantalla = pygame.display.set_mode((800, 600))
pygame.display.set_caption("Eventos del Teclado")

fuente = pygame.font.Font(None, 36)
mensaje = "Presiona una tecla"

corriendo = True
while corriendo:
    for evento in pygame.event.get():
        if evento.type == pygame.QUIT:
            corriendo = False
        
        # Detectar cuando se PRESIONA una tecla
        if evento.type == pygame.KEYDOWN:
            if evento.key == pygame.K_SPACE:
                mensaje = "Presionaste ESPACIO"
            elif evento.key == pygame.K_RETURN:
                mensaje = "Presionaste ENTER"
            elif evento.key == pygame.K_ESCAPE:
                mensaje = "Presionaste ESC"
                corriendo = False
            elif evento.key == pygame.K_UP:
                mensaje = "Flecha ARRIBA"
            elif evento.key == pygame.K_DOWN:
                mensaje = "Flecha ABAJO"
            elif evento.key == pygame.K_LEFT:
                mensaje = "Flecha IZQUIERDA"
            elif evento.key == pygame.K_RIGHT:
                mensaje = "Flecha DERECHA"
            else:
                # Para letras y n√∫meros
                mensaje = f"Presionaste: {pygame.key.name(evento.key)}"
        
        # Detectar cuando se SUELTA una tecla
        if evento.type == pygame.KEYUP:
            mensaje = "Soltaste una tecla"
    
    pantalla.fill((30, 30, 30))
    texto = fuente.render(mensaje, True, (255, 255, 255))
    pantalla.blit(texto, (50, 250))
    
    pygame.display.flip()
    pygame.time.Clock().tick(60)

pygame.quit()
sys.exit()

import pygame
import math

# Funci√≥n para detectar colisi√≥n entre c√≠rculos
def colision_circular(x1, y1, radio1, x2, y2, radio2):
    # Calcular distancia entre centros
    distancia = math.sqrt((x2 - x1)**2 + (y2 - y1)**2)
    # Hay colisi√≥n si distancia < suma de radios
    return distancia < (radio1 + radio2)

import pygame
import sys

pygame.init()
pantalla = pygame.display.set_mode((800, 600))
reloj = pygame.time.Clock()

# Definir clase para el jugador
class Jugador(pygame.sprite.Sprite):
    def __init__(self, x, y):
        super().__init__()
        # Crear una superficie como imagen
        self.image = pygame.Surface((50, 50))
        self.image.fill((0, 255, 0))
        self.rect = self.image.get_rect()
        self.rect.center = (x, y)
        self.velocidad = 5
    
    def update(self):
        # Actualizar posici√≥n
        teclas = pygame.key.get_pressed()
        if teclas[pygame.K_LEFT]:
            self.rect.x -= self.velocidad
        if teclas[pygame.K_RIGHT]:
            self.rect.x += self.velocidad
        if teclas[pygame.K_UP]:
            self.rect.y -= self.velocidad
        if teclas[pygame.K_DOWN]:
            self.rect.y += self.velocidad

# Crear grupo de sprites
todos_sprites = pygame.sprite.Group()
jugador = Jugador(400, 300)
todos_sprites.add(jugador)

corriendo = True
while corriendo:
    for evento in pygame.event.get():
        if evento.type == pygame.QUIT:
            corriendo = False
    
    # Actualizar todos los sprites
    todos_sprites.update()
    
    pantalla.fill((0, 0, 0))
    
    # Dibujar todos los sprites
    todos_sprites.draw(pantalla)
    
    pygame.display.flip()
    reloj.tick(60)

pygame.quit()
sys.exit()
